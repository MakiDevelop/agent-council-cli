## Goal

Create and polish a public `agent-council-cli` repo: a lightweight CLI that runs
the same prompt across multiple already-authenticated AI CLIs, compares their
answers, preserves audit logs, and stays out of auth/token management.

## Completed

- Created and pushed `agent-council-cli`.
- Added `ask`, `continue`, `chat`, `sessions`, `status`, and `watch` commands.
- Added `agents.yaml` / `agents.json` support for custom local agents.
- Documented the auth boundary: users log in to each agent CLI themselves;
  this tool does not manage OAuth, API keys, tokens, or browser login flows.
- Added multilingual README sections: Traditional Chinese, Simplified Chinese,

## Key Decisions

- Keep the core product as “same prompt, multiple local AI CLIs, compare
  answers,” not a workflow orchestrator.
- Do not build tmux panes, worktree management, web UI, task routing, voting,
  persistent memory, or OAuth/token management in the core.
- Treat Karpathy’s `llm-council` as inspiration, not a direct competitor.
- Keep memory as a sidecar:
  - candidate extraction first;
  - human or policy approval before any durable write;
  - memhall and mem0 are export targets, not hard dependencies.
- Public users may use mem0 or another memory store; Maki’s environment can use
  memhall through the `memhall-jsonl` export.
